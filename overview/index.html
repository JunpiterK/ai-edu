FAB 용어집 — 먼저 필요한 말만 잡고 간다

왼쪽은 글을 읽는 데 꼭 필요한 핵심 용어, 오른쪽은 시스템 레이어를 이해할 때 참고할 용어입니다. 암기장이 아니라 읽기 지도처럼 쓰면 됩니다.

① 만드는 것 · 현장

FAB (Fabrication Plant) — 반도체·디스플레이를 만드는 초청정 생산공장.

Wafer / Lot — 회로를 새기는 실리콘 원판 / 그 묶음(보통 25장). 처리·추적의 기본 단위.

Recipe (레시피) — 장비에 주는 공정 조건표(온도·압력·시간·가스 등).

PPID (Process Program ID) — 장비가 실행할 레시피/공정 프로그램의 식별자. 교육 자료에서는 Recipe ID(PPID)로 함께 표기.

Chamber / Equipment — 실제 공정이 일어나는 장비(노광·식각·증착 등).

Metrology / Inspection (계측 / 검사) — 두께·선폭(CD)·정렬을 재는 것 / 결함을 찾는 것.

② 장비 통신 + 데이터 저장

현장에서 모델을 “재학습한다”는 말은 가중치를 다시 조정한다는 뜻입니다. 그래서 데이터가 바뀌면 같은 알고리즘도 새 모델이 됩니다.

남길 질문
  1. 가중치는 무엇을 조절하는 '손잡이'인가?
  2. 손실이 줄어드는 방향은 어떻게 알아내나?
  3. '순전파 → 역전파 → 갱신' 한 바퀴에서 각 단계는?
정답 확인
  1. 각 입력이 출력에 미치는 영향력(중요도)을 조절하는 손잡이 — 입력마다 곱해지는 숫자입니다.
  2. 기울기(gradient) — 손실을 각 가중치로 미분해 어느 방향으로 움직이면 줄어드는지 계산합니다(역전파).
  3. 순전파=현재 가중치로 예측 → 역전파=오차 책임을 뒤로 배분 → 갱신=기울기 반대 방향으로 가중치를 조금 수정.

순전파와 역전파는 한 바퀴의 흐름으로만 기억해도 충분합니다.

CH4 · 알고리즘 지도① CNN — 합성곱

CNN — 이미지의 공간 패턴을 보는 신경망

반도체·디스플레이 공정의 결함 검사(ADC)가 바로 이 계열입니다. 🏭 현장 검증 · ADC 양산 표준

① 어떻게 탄생했나

1980년 후쿠시마의 Neocognitron에서 영감 → 1998년 Yann LeCun, "Gradient-Based Learning Applied to Document Recognition"(LeNet-5). 손글씨 우편번호를 읽히려고, 이미지를 통째로 넣는 대신 작은 필터를 밀며 특징을 뽑게 했습니다.

② 핵심 원리

합성곱(필터 슬라이딩) + 가중치 공유 + 풀링. 같은 필터를 이미지 전체에 재사용합니다. 가중치 공유는 패턴 위치가 바뀌면 특징맵도 같이 움직이는 성질을 만들고, 풀링/전역 집계가 부분적 위치 불변성을 보탭니다.

③ 이후 발전

AlexNet(2012) — ImageNet 우승, ReLU·GPU·드롭아웃으로 딥러닝 붐을 점화 → VGG·ResNet(2015) 잔차연결로 100층+ → ConvNeXt(2022) CNN 구조를 현대화.

④ 현장에서 (fab)

AOI·SEM 결함 이미지 자동 분류(ADC). Applied ExtractAI, KLA eSL10, Intel 전팹 결함분류가 모두 이 계열. (사례 → 9장)

직관"이미지를 작은 창으로 훑어 패턴을 층층이 쌓는다" — 5장에서 직접 돌려볼 수 있습니다.
CH4 · 알고리즘 지도② Transformer

Transformer의 핵심 — 지금 토큰이 참고할 토큰을 고른다

예: “감지”는 “이상”, “속도”, “증착”을 얼마나 참고할까요? Transformer 계열은 문장·이미지 패치·센서 구간을 모두 토큰처럼 다루고, 이 참고 비중으로 문맥을 만듭니다. 🏭 현장 검증 · LLM·RAG 운영 사례의 기반

1. Self-attention 한 번의 흐름

“감지” 토큰은 질문(Q)을 만들고, 각 토큰의 검색 태그(K)와 비교합니다. 점수가 높은 단어일수록 더 많이 참고합니다.

Q질문: 무엇을 참고할까?K검색 태그: 나는 어떤 의미인가?V가져와 섞을 실제 내용

2. 점수를 참고 비중으로 바꾸기

Attention(Q,K,V)
= softmax(QKᵀ / √dₖ) V

softmax는 점수를 합이 1인 참고 비중으로 바꾸고, 마지막 V는 그 비중대로 정보를 섞는 단계입니다.

3. 확장: 여러 head와 GPT 방식

MULTI-HEAD여러 기준으로 반복, 출력은 이어붙여 섞음GPT미래 토큰을 가리고 다음 토큰 예측TEXT문서 chunk에서 근거 문장 검색TRACE센서 구간 간 drift 전후 연결

4. Fab 적용 시 해석 기준

문서는 어떤 SOP·로그 chunk를 봤는지, 이미지는 어떤 패치 주변을 봤는지, 센서는 어떤 trace 구간을 봤는지 확인합니다. 단, attention 가중치는 원인 증명이 아니라 검토할 단서입니다.

SELF-ATTENTION같은 입력 안에서 서로를 조회문장·이미지·센서 구간 내부 관계를 한 번에 계산합니다. MULTI-HEAD여러 관점을 동시에 본다연관 단서, 시간 순서, 문맥 관계, 수치 패턴을 다른 head가 나눠 봅니다. CAUSAL LLM미래 토큰을 가린다GPT 계열은 이전 문맥만 보고 다음 토큰 분포를 맞힙니다. CHECKPOINTAttention은 단서, 검증은 별도가중치가 높다고 곧바로 원인이라는 뜻은 아닙니다.
CH4 · 알고리즘 지도③ 시계열 (RNN·LSTM)

시계열 — 과거의 흐름으로 미래를 예측

센서 시계열로 막두께를 추정하는 가상계측(VM)의 토대. 🏭 현장 검증 · FDC·PdM 적용 표준

① 어떻게 탄생했나

순환신경망(RNN)은 앞 정보를 기억하려 했지만 길어지면 기억이 사라졌습니다(기울기 소실). 1997년 Hochreiter & Schmidhuber, "Long Short-Term Memory(LSTM)"가 게이트로 이 문제를 풀었습니다.

② 핵심 원리

셀 상태 + 3개 게이트(망각·입력·출력). 무엇을 지우고/새로 담고/내보낼지 게이트가 조절해 중요한 정보를 오래 유지합니다.

③ 이후 발전

간소화한 GRU(2014) → 이후 Transformer 계열 PatchTST(2023)·TimesFM(2024)이 긴 문맥·다변량 예측에서 강력한 후보로 등장했습니다.

④ 현장에서 (fab)

가상계측(VM)으로 측정 안 한 wafer 값 추정, 센서 수요·드리프트 예측, 예지보전(PdM). (VM 해부 → 8장)

직관"기억할 건 남기고 잊을 건 버리는 메모장" — 5장 LSTM 게이트 시뮬에서 c(기억)가 유지되는 걸 직접.
CH4 · 알고리즘 지도④ Tabular — 트리 부스팅

그래디언트 부스팅 — 틀린 만큼만 다시 배우는 트리들의 릴레이

잔차(남은 오차)를 다음 트리가 이어받아 조금씩 고친다 — XGBoost·LightGBM이 정형(표) 데이터의 사실상 표준인 이유 🏭 현장 검증 · 수율 분석 표준(+SHAP)

계보 — 각 세대는 앞 세대의 약점을 고쳤다

CART 1984(단일 트리, 과적합 취약) → AdaBoost 1997(틀린 샘플에 가중치, 약한 학습기 결합) → GBM 2001 Friedman(임의의 손실함수를 ‘잔차 = 음의 그래디언트’로 일반화) → XGBoost 2016 Chen&Guestrin(정규화 + 2차 근사 + 병렬화)·LightGBM 2017 MS(히스토그램·leaf-wise로 대용량 고속화) → SHAP 2017 Lundberg&Lee(예측 근거의 정량 설명).

왜 표(Tabular) 데이터의 왕인가

신경망은 매끈한 함수를 선호해 단위·분포가 제각각인 표 데이터에 불리하다(Grinsztajn et al. 2022 벤치마크). GBT는 결측치 자동 처리·스케일링 불필요·비선형 상호작용 자동 포착·수천 행 소량 데이터에서도 강해, 캐글 정형 데이터 대회 우승 솔루션의 단골이 됐다.

핵심 하이퍼파라미터 — 세 개의 손잡이

트리 수(n_estimators): 잔차를 고치는 횟수. 학습률(learning_rate): 한 번에 고치는 폭 — 작을수록 안정적이지만 트리가 더 필요. 깊이(max_depth): 한 트리가 잡을 수 있는 feature 상호작용의 차수(깊을수록 복잡·과적합 위험).

팹 적용 — 불량 원인 후보 랭킹 + SHAP

WAT·센서·설비 로그 수백 개 feature로 수율/불량 확률을 예측하고, SHAP로 웨이퍼 한 장 단위의 기여도를 읽는다 — “이 로트는 챔버 압력 feature가 불량 확률을 +18%p 밀어올렸다”처럼 원인 후보가 랭킹으로 나온다. 수식·코드는 별책부록 XGBoost 딥다이브에서 이어진다.

직관GBT는 한 명의 천재가 아니라 앞사람의 오답노트만 공부하는 수백 명의 평범한 학생이다 — 5장 GBT·SHAP 시뮬레이터에서 학습률·트리 수를 직접 조작하며 잔차가 줄어드는 과정을 확인할 수 있다.
CH4 · 알고리즘 지도⑤ GNN — 그래프

GNN — 관계가 데이터일 때, 이웃의 상태를 모아 나를 갱신한다

장비·챔버·lot처럼 '연결' 자체가 정보인 데이터에서, 메시지 패싱 라운드가 이상 신호를 hop 단위로 퍼뜨린다 🎓 학계 SOTA·확산 중 · Intel·ASU 연구 단계

① 왜 그래프인가 — 표로 펴면 관계가 사라진다

이미지는 격자(CNN), 문장은 순서(RNN·Transformer), 팹 데이터는 연결이다. 장비–챔버–lot–공정 step은 "이 lot이 어느 챔버를 거쳤나"라는 관계가 본질인데, 표(행렬)로 펴는 순간 그 연결 정보가 사라진다. GNN은 노드(개체)와 엣지(관계)를 그대로 입력으로 받는다.

② 메시지 패싱 3단계 — 수집 → 집계 → 갱신

각 노드가 이웃의 현재 값을 메시지로 수집하고, 순서와 무관한 함수(평균·합·max)로 집계한 뒤, 자기 값과 결합해 갱신한다(Gilmer 2017 프레임워크, distill.pub 해설). 이 라운드를 k번 반복하면 k-hop 이웃까지의 정보가 내 표현에 스며든다 — 라운드 수 = 정보 도달 거리.

③ 계보 — GCN → GraphSAGE → GAT

GCN(Kipf & Welling, ICLR 2017)은 이웃을 차수 정규화된 평균으로 집계 — 모든 이웃을 동일 취급. GraphSAGE(2017)는 이웃을 샘플링하고 집계 함수를 학습해 처음 보는 노드에도 일반화. GAT(Veličković, ICLR 2018)는 어텐션으로 이웃마다 다른 가중치를 학습해 중요한 이웃에 집중한다.

④ 팹 적용 — Intel Foundry·ASU의 GAT 가상계측 연구

공정 step×parameter 쌍을 노드로 두고, 파라미터→막층(layer) 의존성을 어텐션으로 집계해 증착 막두께를 예측(arXiv 2026, 연구 단계) — 학습된 어텐션 가중치가 실제 공정 물리와 일치했다. 일반적 활용: 다장비 영향 추적, 수율 저하 lot 클러스터 탐지. 상세는 별책부록 gnn.

직관내 상태 = 이웃 상태의 요약과 나의 결합 — 라운드를 거듭할수록 이상 신호가 소문처럼 그래프를 타고 퍼진다. 5장 GNN 메시지 패싱 시뮬레이터에서 라운드를 직접 돌려보며 hop별 전파를 확인한다.
CH4 · 알고리즘 지도⑥ 강화학습 (RL)

강화학습 — 정답 대신 보상으로 배우는, 행동하는 AI

시행착오(trial-and-error)와 지연 보상(delayed reward) — Sutton & Barto가 꼽은 RL의 두 가지 본질. 에이전트가 환경과 상호작용하며 "무엇을 할지"를 스스로 익힌다 🎓 학계 SOTA·확산 중 · 시뮬·오프라인 연구 단계

지도학습과 뭐가 다른가 — 세 가지

① 정답 라벨이 없다 — "잘했다/못했다"는 보상 숫자 하나뿐. ② 보상이 늦게 온다 — 수율 결과는 수십 스텝 뒤에 도착, 어느 행동 덕분인지 스스로 추적해야 한다(credit assignment). ③ 내 행동이 다음 데이터를 바꾼다 — 데이터가 고정된 지도학습과 달리, 정책이 바뀌면 마주치는 상태 분포 자체가 바뀐다.

탐험 vs 활용 — RL의 숙명, 그리고 안전 문제

아는 최선만 반복하면(활용) 더 나은 수를 영영 못 찾고, 새 시도만 하면(탐험) 성과가 없다. ε-greedy는 "90%는 최선, 10%는 무작위 시도"로 균형을 잡는 가장 단순한 답. 문제는 팹에서 이 '무작위 시도' 자체가 웨이퍼 폐기·장비 사고가 될 수 있다는 것 — RL 실전 적용의 최대 장벽이다.

계보 — 표 하나에서 세계 챔피언까지

Q-러닝 1989(Watkins) 표에 행동 가치 기록 → DQN 2015(DeepMind) 표를 신경망으로 대체, 아타리 게임 정복 → AlphaGo 2016 자기대국으로 이세돌 격파 → PPO 2017 안정적 정책 학습, 현재 LLM의 RLHF 표준 엔진 → 오프라인 RL·CQL 2020~ 시행착오 없이 과거 로그만으로 보수적 학습.

팹 현실 — 시뮬레이터 안에서 크는 중

실장비 시행착오는 위험·비용 때문에 불가 → 디지털트윈·시뮬레이터 학습, 오프라인 RL이 중심인 연구 단계. 유력 후보는 로트 디스패칭·스케줄링과 APC 파라미터 튜닝이지만, 문헌 대부분이 시뮬레이션 검증에 머물러 상용 검증 사례는 아직 드물다. Lam의 Nature 2023 식각 레시피 연구도 RL이 아닌 베이지안 최적화였다. 별책부록 qlearn·ppo·cql

직관RL은 정답지를 보고 배우는 학생이 아니라, 점수판만 보고 스스로 수를 찾아가는 게임 플레이어다 — 빈 Q-표가 한 칸씩 채워지는 과정은 5장 Q-러닝 시뮬레이터에서 직접 돌려본다.
CH4 · 알고리즘 지도⑦ Diffusion — 생성

Diffusion 생성 모델 — 노이즈를 걷어내며 이미지를 만든다

깨끗한 이미지에 노이즈를 조금씩 더하는 과정은 쉽다. 그 역과정을 신경망에 학습시키면, 순수 노이즈에서 진짜 같은 이미지가 태어난다. 🎓 학계 SOTA·확산 중 · 데이터 증강 연구 단계

1핵심 아이디어 — 어려운 문제를 쉬운 문제로

이미지에 노이즈를 더하는 전방 과정은 계산이 쉽다. 그러므로 그 역과정만 신경망에 배우게 한다.

학습 목표는 단 하나: “이 스텝에 낀 노이즈 ε가 무엇인가”를 맞히는 것(ε-예측). 잘 맞히면 노이즈를 걷어내 원본을 복원할 수 있다.

2왜 GAN보다 안정적인가

적대적 경쟁이 없어 모드 붕괴가 거의 없고, 데이터 분포를 폭넓게 덮어 다양성·학습 안정성이 높다.

대신 수백~수천 스텝을 밟아 샘플링이 느린 것이 약점 → DDIM·증류·ODE 계열 가속 연구로 10~50스텝까지 단축.

3계보 — 한눈에

DDPM(2020) 기초 정립(ε-예측) → DDIM 결정론적 샘플링으로 스텝 대폭 축소.

Latent/Stable Diffusion(2022) 잠재공간 확산 → 고해상도를 소비자 GPU로. DALL·E 2도 확산 기반.

이후 영상·3D로 확장(시간축·3D U-Net 추가).

4팹 적용 — 희귀 결함 데이터 증강

결함이 부족·불균형할 때 희귀 결함 SEM/웨이퍼 이미지를 합성해 검사 모델 학습을 보강(연구 단계, 유형·위치 조건부 생성). 공정 시뮬레이션 응용도 시도.

주의 합성 데이터는 만능이 아니다 — 반드시 실검사 데이터로 분포 검증을 거쳐 분포 밖 인공물 학습을 막아야 한다.

CH4 · 알고리즘 지도⑧ LLM

LLM — "다음 토큰 예측"을 극한으로 키우면 언어 능력이 나온다

사전학습으로 지식을 압축하고, SFT로 지시를 가르치고, RLHF로 사람 선호에 정렬한다 — 그리고 환각은 이 구조의 필연적 부산물이다 🏭 현장 검증 · TCL CSOT·BOE 공개 사례

"다음 토큰 예측"이 왜 지능처럼 보이나

다음 단어를 잘 맞히려면 문법·사실·논리·도메인 지식까지 전부 동원해야 한다. 즉 예측 훈련은 방대한 텍스트의 패턴을 가중치에 압축하는 과정이다. 스케일링 법칙: 데이터·파라미터·연산량을 키우면 성능이 멱법칙을 따라 예측 가능하게 좋아진다 — "크게 만들수록 좋아진다"가 GPT 시대를 연 발견이다.

3단계 만들기 — 각 단계의 역할

사전학습은 지식과 언어 능력을 넣고, SFT는 "질문에 답하는 형식"을 가르치고, RLHF는 사람이 선호하는 응답 방향으로 정렬한다. InstructGPT 실험에서는 RLHF를 거친 1.3B 모델이 175B 원본보다 사람 평가에서 선호됐다 — 크기만큼 정렬이 중요하다는 증거다.

환각의 원인과 대책

LLM은 사실 데이터베이스가 아니라 확률적 이어쓰기 기계다. 훈련 데이터에 없거나 애매한 내용도 가장 그럴듯한 토큰을 이어붙이므로, 틀린 답도 유창하게 나온다. 대책은 세 겹: 근거 검색(RAG)으로 사내 문서를 붙이고, 출처 표시로 검증 가능하게 하고, 중요한 판단은 사람이 검토한다.

팹 적용 — 검증된 사례와 현실적 쓰임새

TCL CSOT X-Intelligence 3.0(32B): 디스플레이 특화 추론 LLM — 이슈 분석 20%·소재 개발 30% 효율 개선을 발표(DTC 2025). BOE Blue Whale: 제조·운영 전반에 적용된 디스플레이 특화 LLM. 일반적으로는 SOP·로그 검색, 신입 교육이 출발점이며, 자동 판단이 아니라 근거+권한+검토가 붙을 때 실무 도구가 된다.

직관LLM은 "다음 토큰 예측"을 압도적 규모로 키워 언어 능력을 얻은 모델이다 — 그럴듯함이 본질이므로 팹에서는 반드시 근거(RAG)를 붙여 쓴다. 자세한 활용은 7장 LLM·RAG, 원리는 별책부록 llm 딥다이브에서 다룬다.

"다음 단어" 예측기와 "의미의 좌표"

단어를 좌표(벡터)로. 의미가 비슷하면 각도가 작다.

LLM = "다음 단어" 예측기

앞 내용을 보고 다음에 올 단어의 확률을 예측하고, 그걸 반복해 문장을 만듭니다. "이해"처럼 보이지만 본질은 방대한 패턴 예측입니다.

임베딩 = 의미의 좌표

단어·문장을 숫자 좌표(벡터)로 바꿉니다. 의미가 가까우면 좌표도 가까워, "비슷한 문서 찾기"가 거리(각도) 계산이 됩니다 — 이게 RAG 검색의 엔진.

예"수율이란 ___" 다음에 "양품의 비율"이 올 확률이 높게 학습돼 있으면 자연스럽게 이어 씁니다. 하지만 학습·문맥에 없는 조직 내부 문서는 정확히 알 수 없어 → RAG가 필요합니다.

핵심 요약
  • 데이터 함정(라벨·드리프트·불균형·누설)과 운영 함정(섀도우·MLOps·검토·IP).
  • SOTA ≠ 양산 적합 — 불확실성·안전이 먼저.
  • 현장 LLM은 보통 RAG + 권한관리 + 사람 검토(사인오프)가 필요합니다.
최종 기준

배포 가능한 AI는 예측값보다 운영 절차가 선명합니다

모델이 틀렸을 때 누가 보고, 어떻게 hold하고, 어떤 로그로 원인을 추적할지까지 있어야 현장 적용입니다.

남길 질문
  1. '섀도우 배포'가 왜 필요한가?
  2. 현장 LLM 도입의 3요소는?
  3. 진짜 자산은 '모델 1개'가 아니라 무엇인가?
정답 확인
  1. 실제 운영에 영향을 주지 않고 예측을 로그로만 검증 — 배포 전에 현장 성능과 리스크를 확인하기 위해서입니다.
  2. 근거(RAG·출처) · 권한(접근 통제) · 검토(사람 승인 루프).
  3. 데이터 계약·검증 체계·운영 루프(MLOps) — 모델은 갈아끼워도 이 기반이 조직의 자산으로 남습니다.

운영 리스크를 통과하지 못한 AI는 성능이 좋아도 배포되지 못합니다.
