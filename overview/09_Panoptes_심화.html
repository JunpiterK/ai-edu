① 무엇이 공개되었나사실 vs 추측

Panoptes — 외부에서 확인 가능한 것만

이름의 어원

그리스 신화 판옵테스(100개의 눈을 가진 거인) — "모든 wafer를 다 본다"는 VM의 지향을 담은 작명.

검증 가능  공개 사실

SK hynix가 자체 VM 시스템을 Panoptes로 부르며 운영한다고 공개 발표·기사에서 언급. "계측을 줄이고 AI로 추정"이라는 방향성.

부분 공개  개략 정보

VM이 수율·생산성에 기여한다는 정성적 언급 — 구체 수치·구조는 대부분 비공개.

추측 금지비공개 모델 구조·정확도 수치·센서 목록·학습 규모는 공개되지 않았습니다. 이 자료는 그 영역을 지어내지 않습니다.
그래서 이 자료가 가르치는 것
Panoptes라는 이름을 빌려, VM이 일반적으로 어떻게 동작하는가(학술·산업 표준)를 배웁니다. 특정 회사의 비밀이 아니라 VM이라는 기술 그 자체.
② 왜 VM인가측정 부담

왜 VM이 필요한가 — 측정의 두 손실

현실

전수 계측은 불가

한 lot의 모든 wafer를 다 재면 느리고 비쌈. 그래서 일부만 표본 측정.

손실 ①

못 본 wafer

표본 사이의 wafer는 측정값이 없음 → 이상이 늦게 발견.

손실 ②

지연된 피드백

계측 대기 동안 같은 오차가 계속 누적 → 제어가 늦음.

VM의 해법공정 센서값으로 매 wafer의 측정값을 즉시 추정 → 측정 안 한 wafer도 추정치 확보, 피드백(APC)을 빠르게. 미세화로 공차가 줄수록 이 압력은 커집니다.
③ 일반 아키텍처학술 표준

VM 일반 데이터 플로우 — 학술 표준 구조

[장비 센서/FDC] ── 공정 중 신호 ↓ 특징 추출(요약 통계) 특징 벡터 (수십~수백) ↓ [VM 모델] ── 신경망/트리/회귀 ↓ 추정 측정값 + 신뢰도 ↓ ┌───────────┴───────────┐ [APC 피드백] [실측과 비교 시 즉시 보정 → 모델 보정]

입력 — wafer 한 장의 흔적

압력·온도·RF·가스유량 시계열을 요약 특징으로. wafer마다 고유한 "공정 지문".

출력 — 추정값 + 신뢰도

두께 등 추정치 하나만이 아니라 "얼마나 믿을만한가"를 함께 — 이게 운영의 핵심.

폐루프 — 가끔의 실측으로 보정

드물게 실제 계측과 비교해 드리프트를 추적하고 모델을 갱신.

④ 알고리즘Calibration

신뢰도 보정 — VM에서 진짜 어려운 부분

왜 정확도보다 신뢰도?

VM 추정으로 실측을 건너뛸지를 결정해야 함. "이 추정 믿어도 되나"가 틀리면 불량을 놓침 → 신뢰도 자체가 정확해야.

보정(Calibration)이란

"90% 확신"이라고 말한 예측이 실제로 90% 맞아야 보정된 모델. 과신·과소신을 교정.

운영신뢰도 높으면 실측 생략, 낮으면 실측으로 회부 — 측정 부담을 줄이면서 안전을 지키는 손잡이.
예측: 두께 = 21.3 nm 신뢰도: 0.94 → 실측 생략 OK 예측: 두께 = 19.8 nm 신뢰도: 0.61 → 실측으로 회부 # 보정 점검 "0.9 확신" 예측들 중 실제 적중률이 ~0.9 인가? YES → 신뢰 가능한 손잡이 NO → 재보정 필요
★ 캘리브레이션 LIVE보정된 모델 vs 과신 모델

신뢰도 곡선 — "90% 확신"이 정말 90% 맞는가

읽는 법

점이 대각선 위면 = 말한 신뢰도 = 실제 적중률 → 보정됨. 대각선 아래로 처지면 = 과신(말은 90%인데 실제 70%).

운영 영향—

과신 모델은 "믿어도 된다"고 잘못 말해 실측을 생략시키고 → 불량을 놓칩니다. 그래서 정확도보다 보정이 먼저.

⑤ 운영wafer 1장의 여정

wafer 1장이 VM을 통과하는 일반 흐름

1

공정 + 수집

장비가 처리하며 센서 신호를 실시간 기록.

2

추정

특징 추출 → 모델이 측정값 + 신뢰도 출력 (ms).

3

분기

신뢰도↑ → 통과 / 신뢰도↓ → 실측 회부.

4

피드백·보정

추정으로 APC 보정, 실측 들어오면 모델 갱신.

효과측정 안 한 wafer도 추정치 확보 → 사각지대 축소 + 빠른 피드백. 이것이 VM(그리고 Panoptes 같은 시스템)이 노리는 가치입니다.
