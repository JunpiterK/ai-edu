★ CPU vs GPU LIVE같은 행렬곱, 누가 빠른가

행렬곱 6×6 — CPU는 한 칸씩, GPU는 여러 칸을 동시에

CPU · 코어 소수 (순차)
클럭 0 · 완료 0/36
GPU · 코어 12개 (병렬)
클럭 0 · 완료 0/36

왜 차이가 나나

출력 C의 각 칸은 독립적인 내적 — 서로 안 기다림. GPU는 이 칸들을 수천 갈래로 동시에 계산.

이게 "병렬"의 본질

행렬곱이 곧 병렬 가능 → 학습 시간을 며칠 → 몇 시간으로. Tensor Core는 더 가속.

결과를 기다리는 중…
GPU 코어 수:

CPU는 36클럭(한 칸씩), GPU는 코어 수만큼 나눠 동시에 — 코어 36이면 단 1클럭. 실제 칩은 코어가 수천 개.

② 분업학습·추론·엣지

같은 모델의 3가지 다른 운명

학습 (Train)

한 번, 아주 무겁게

역전파 포함 — 가장 무거움. GPU/TPU 클러스터에서 며칠~몇 주. 메모리·인터커넥트가 병목.

클라우드 추론

매일, 대량으로

순전파만. 처리량·비용·지연이 관건 → GPU/TPU + INT8 양자화로 효율화.

엣지 추론

기기 안에서, 즉시

네트워크 없이 NPU가 저전력으로. 프라이버시·지연·오프라인이 강점.

의사결정 프레임 — "어떤 칩?"
① 학습인가 추론인가 → ② 지연·프라이버시가 기기 내 처리를 요구하나 → ③ 전력 예산은 → ④ 생태계/이식성. 이 순서로 좁히면 답이 거의 정해집니다.
③ LLM학습·추론·엣지

LLM 하드웨어 — 거대 학습부터 폰까지

단계하드웨어특징
학습수천~수만 GPU/TPU 클러스터수 주, 고속 인터커넥트가 핵심 (GPT·Gemini·Claude 등)
클라우드 추론GPU/TPU + 양자화·서빙 최적화KV캐시·배칭으로 처리량↑·지연↓
엣지 추론폰/노트북 NPU (소형·양자화 모델)오프라인·프라이버시. 작은 모델이 기기 내에서 직접 구동
현장 온프레온프레미스 GPU 서버 + 오픈웨이트망분리·데이터보안 (gpt-oss 등) → 10·RAG 자료
경향"학습은 점점 더 크게, 실행은 점점 더 작게·가까이" — 양자화·증류로 같은 능력을 더 작은 칩에서.
④ Physical AI로봇 · VLA

로봇 — 보고 · 듣고 · 행동하는 VLA

VLA = Vision-Language-Action

카메라 영상 + 언어 지시 → 행동(관절 명령)을 직접 출력하는 모델. "컵을 집어"를 보고 바로 움직임으로.

이중 구조 — 학습 vs 실행

거대 모델은 클라우드에서 학습, 로봇 안에서는 NPU가 실시간 추론. 지연이 안전과 직결.

탑재 칩로봇용 NPU의 사실상 표준은 NVIDIA Jetson 계열 — 저전력으로 비전+제어를 한 보드에.
[카메라] [마이크/지시문] ↓ 인코더(비전+언어) 공통 표현 ↓ 정책(Transformer) [관절 토크 · 그리퍼 명령] ↓ 실행 → 다시 관찰 (폐루프) # 학습: 클라우드 GPU 클러스터 # 실행: 로봇 내 NPU(Jetson 등) # 핵심 = 저지연 · 저전력
⑤ 자율주행차량용 칩

Tesla FSD

Mobileye

비전 중심의 전용 SoC로 ADAS~자율주행. 효율적 저전력 인식.

공통점셋 다 차량 내 저전력·실시간 추론(NPU/전용 SoC) + 클라우드 대규모 학습의 조합. 로봇과 같은 "이중 구조".

제조사별 칩 명칭·세대·성능은 빠르게 갱신됩니다 — 접근 방식의 차이로 읽어 주세요.
