생성형 AI 현장 활용

2개 글

제조 문제는 잘 알지만 개발자로 출발하지 않은 엔지니어를 위한 현장 기록입니다. 개인적인 사용 경험과 공식 문헌으로 확인한 기능을 분리하고, 보안 경계를 먼저 설명합니다.

이 시리즈의 원칙

반복되는 작은 업무 하나에서 시작해 입력과 합격 조건을 적고, 승인된 환경으로 들어가기 전에는 가상 데이터만 사용합니다. 생산성보다 제조 노하우 보호가 먼저입니다.

  1. 현장 AI경험 + 문헌

    비개발 FAB 엔지니어가 생성형 AI를 업무 도구로 만든 방법

    ChatGPT, Gemini, Grok, Claude를 약 2년간 사용한 경험과 오프라인 Python 의존성, LTE 태블릿 편집, 보안 경계, 공식 기능 비교표를 담았습니다.

  2. Mock 랩실행형 실습

    GPU 없이 AI 에이전트 운영 경계를 검증하는 Mock 랩

    저사양 PC에서 검색 권한, 도구 계약, 승인 바인딩, 감사 로그 장애, 재사용 차단, 반복 한도를 8개 무의존성 테스트로 확인합니다.
